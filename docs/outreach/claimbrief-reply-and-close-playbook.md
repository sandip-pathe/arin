# ClaimBrief Reply And Close Playbook

Use this after the first outreach batch goes out. The goal is not "learn more."
The goal is to get one real redacted packet, return one useful sample, and ask
for payment.

## Rule

Every positive reply should move to one of three asks:

1. Send one old closed/redacted claim packet.
2. Book a 15-minute review only after they send a packet.
3. Pay for the starter pilot after the sample is delivered.

Do not do unpaid discovery calls before seeing a real packet unless the prospect
is a high-value firm with clear volume.

## Reply Buckets

| Reply Type | Meaning | Next Move |
| --- | --- | --- |
| "Sure, send info" | Mild interest | Send sample link and ask for one redacted packet. |
| "Can you show an example?" | Wants proof | Send sample packet link and restate free sample offer. |
| "We have a file" | Strong signal | Ask for redacted/closed packet and set 24-hour expectation. |
| "How much?" | Buying signal | Give pilot pricing and still offer one free sample. |
| "Is this legal advice?" | Trust risk | State boundaries clearly. |
| "We use X / have a CRM" | Category confusion | Say this is not CRM; it is a file briefing artifact. |
| No reply | Normal | Follow up once after 48 hours, then once after 5 business days. |

## Positive Reply: Send The Packet

```text
Great. Please send one old closed or redacted claim packet.

Packet instructions:
https://app.anaya.legal/claimbrief/intake

Best packet:
- policy excerpts or full policy
- carrier denial/coverage letter
- carrier estimate
- contractor/public-adjuster estimate if available
- key correspondence

I will return:
- cited review packet
- missing evidence checklist
- draft response outline for human review
- PDF/Markdown export

Target turnaround: 24 hours.
```

## Legal Advice Concern

```text
Good question. ClaimBrief is not legal advice and does not provide public-adjusting services.

It does not:
- contact carriers
- submit claims
- negotiate claims
- tell policyholders what to accept
- make coverage determinations

It only organizes the documents into a cited internal review packet for a licensed professional to review, edit, or ignore.
```

## CRM Objection

```text
Totally fair. This is not a CRM.

ClaimBrief is closer to "brief this messy claim file before I spend an hour reading it."

It outputs a PDF/Markdown review packet: denial reasons, policy provisions, missing evidence, and a response outline. Your existing CRM can still be the system of record.
```

## Security / Confidentiality Concern

```text
For the first sample, please use an old closed or redacted packet.

Remove anything you do not want reviewed. I will use the file only to create the sample ClaimBrief and will not share it, contact anyone, or use it in marketing.

If the sample is useful and you want to continue, we can set a stricter workflow before any live files are used.
```

## First-Sale Success Criteria

The wedge is alive if the first 150 prospects produce at least one of:

- 2 real redacted/closed claim packets.
- 1 paid $99 starter batch.
